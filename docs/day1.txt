ZendFrameworkのインストール
    プロジェクトで利用するJobeet-For−ZendFrameworkディレクトリを作ります。
        $ mkdir -p /home/zfprojects/jobeet-for-zf/
        $ cd /home/zfprojects/jobeet-for-zf/
    Windowsの場合
        c:\> mkdir c:\development\zfprojects\jobeet-for-zf\
        c:\> cd c:\development\zfprojects\jobeet-for-zf\
    * 注意 *
        Windowsユーザーの方はプロジェクトフォルダを作る際に、スペースが含まれないパスでセットアップすることをおすすめします。

ZendFrameworkのバージョンを選ぶ
    このチュートリアルでは1.10系の利用を想定しています。

ソースのダウンロード
    SVN（推奨）：svn checkout http://framework.zend.com/svn/framework/standard/trunk
    圧縮ファイル：http://framework.zend.com/download/latest

ZendFrameworkのインストール位置を選ぶ
    ZendFrameworkではCLIからプロジェクトを新しく作る際に、「library」というディレクトリが作成されます。
    ZendFrameworkのコアや、別なライブラリを管理するためのディレクトリです。
    ZendFrameworkではこのディレクトリ以下でライブラリを管理することが進められていますので、この「library」ディレクトリ以下にZendFrameworkコアのインストールを行ないます。
    ディレクトリ作成後にlibraryディレクトリに移動しておきます。
    $ mkdir library
    $ cd library

    Subversionからインストールする
        $ svn checkout http://framework.zend.com/svn/framework/standard/trunk/library/Zend/ .
    アーカイブからインストールする
        ダウンロードページから1.10系ソースコードをダウンロードします。
        $ tar xvzf ZendFramework-1.10.x-minimal.tar.gz
        $ mv ZendFramework-1.10.x-minimal/library/Zend/ .
        $ mv ZendFramework-1.10.x-minimal/bin/ .
        $ rm -Rf ZendFramework-1.10.x-minimal ZendFramework-1.10.x-minimal.tar.gz
    * お知らせ *
        文中に出てきたxはマイナーバージョン番号を指ます。　ダウンロードしたファイルによって変動している部分なので、適宣読み替えてください。

インストールの検証
    ZendFrameworkが正常にインストールされているか確認を取ります。
    $ php bin/zf.php show version
      Zend Framework Version: 1.10.x
    このような表示がされればインストールが正常に完了しています。

プロジェクトの作成
    zfprojects/jobeet-for-zfディレクトリから実際のZendFrameworkプロジェクトを作る create projectプロバイダを実行します。
    $ php bin/zf.php create project .
     Creating project at /home/zfprojects/jobeet-for-zf
     Note: This command created a web project, for more information setting up your VHOST, please see docs/README
    これでプロジェクトの作成が完了しました。

    create projectプロバイダはZendFrameworkプロジェクトに必要なディレクトリとファイルのデフォルト構造を生成します。
        application/ プロジェクトのすべてのアプリケーションを格納する
        application/configs/ プロジェクトの設定ファイル
        application/controllers/ モジュールを利用しないプロジェクトで利用するコントローラクラスを格納する
        application/models/ モジュールを利用しないプロジェクトで利用するモデルクラスを格納する
        application/view/ モジュールを利用しないプロジェクトで利用するビューデータを格納する
        docs/ プロジェクトのドキュメントを格納する
        library/ プロジェクトで利用するライブラリを格納する
        public/ プロジェクトのWeb公開ディレクトリ
        tests/ ユニットテストと機能テストのディレクトリ

    zf.phpへのショートカットを設定することで、長ったらしい文字を書かなくてすみます。
    予め、bin/zf.shのパーミッションを0777にしておいてください。
        $ ln -s bin/zf.sh zf

サーバー設定を行う
    実際にZendFrameworkで作成したプロジェクトを動かすには、次のような記述をhttpd.confに記述してください。
        NameVirtualHost jobeet-for-zf:80
        <virtualhost jobeet-for-zf:80>
            ServerName jobeet-for-zf
            DocumentRoot "/home/zfprojects/jobeet-for-zf/public"
            DirectoryIndex index.php
            <Directory>
                AllowOverride All
                Allow from All
            </Directory>
        </virtualhost>
    * 注意 *
        この設定はバーチャルホストの設定なので、より強固なセキュリティを実装する場合は別途apacheのhttpd.confファイルに記述してください！
